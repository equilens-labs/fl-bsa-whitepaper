
\section{Reproducibility}
Every table in this paper is tied to a run identifier and a manifest. The manifest records the dataset hash, code commit, container image digests, random number generator seed, hardware description, and start/end timestamps, as summarised in Appendix~E.

In addition, the whitepaper intake bundle includes an unsigned certificate chain (JSON) for auditability and tamper-evidence; \path{intake/pack_intent.json} records \path{certificate_signing_expected=false}. When certificate signing is enabled for GOLD evidence bundles, certificates are ECDSA P-256 signed and record signature material in fields such as \path{certificate_signature}, \path{signature_algorithm}, and \path{public_key_fingerprint}. For cryptographically verifiable certificate signatures, reviewers should use the GOLD evidence bundles and certificate-chain outputs rather than this whitepaper intake bundle.
