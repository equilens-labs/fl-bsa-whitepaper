
\section{Model and Algorithm}
FL-BSA uses a dual-branch tabular generator to produce synthetic borrower portfolios from tabular lending data. In this whitepaper, fairness is evaluated \emph{post hoc} from aggregate selection counts and does not rely on training-time fairness constraints. The stable v5.0.0 native/no-CTGAN evidence run uses the first-party evidence-native generator. Branch generator configuration and hyperparameter disposition for this run are summarised in Appendix~F.

The amplification (bias-preserving) branch preserves protected-group proportions and historical decision signals so reviewers can test whether the evidence path detects the status quo. The intrinsic (de-biased) branch excludes outcome aliases before generation and then applies the configured post-label policy, creating a counterfactual comparison slice for the same protected-attribute surface. In the system source of truth, this is the core audit question: ``the gap between branches is the primary audit object.'' For this run, the gap is read through the historical, amplification, and intrinsic gender AIR slices reported in \Cref{tab:gender_air_slices,fig:gender_air_slices}.
